# تثبيت Alusus على لينكس وتشغيل Hello World

يشرح هذا الدليل طريقة بسيطة لتثبيت الوسس على لينكس ثم اختبارها ببرنامج Hello World صغير.

## المتطلبات

بحسب صفحة التنزيل الرسمية، توفر الوسس حزم تثبيت للأنظمة الحديثة، ومنها:
- Ubuntu 20.04 أو أحدث
- توزيعات لينكس حديثة أخرى تدعم نوع الحزمة المناسب

## الخطوة 1: تنزيل الحزمة

اذهب إلى صفحة التنزيل الرسمية واختر الحزمة المناسبة لنظامك.

الأنواع الشائعة:
- `.deb` لأنظمة Debian/Ubuntu
- `.rpm` لأنظمة Fedora وRHEL وما يشابهها

## الخطوة 2: تثبيت Alusus

### Debian/Ubuntu

```bash
sudo apt install ./alusus_<version>_amd64.deb
```

### أنظمة RPM

```bash
sudo dnf install ./alusus-<version>.x86_64.rpm
```

تشير الوثائق الرسمية إلى أن ملفات الوسس تُثبّت تحت `/opt/Alusus`، كما تُنشأ روابط تنفيذية تحت `/usr/local/bin`.

## الخطوة 3: التحقق من التثبيت

اختبر توفر الأمر:

```bash
alusus --help
```

إذا لم يعمل، فتحقق من وجود الملفات في:
- `/usr/local/bin`
- `/opt/Alusus`

## الخطوة 4: إنشاء ملف Hello World

أنشئ ملفًا باسم `hello.alusus`:

```alusus
import "Srl/Console.alusus";
use Srl;

Console.print("Hello World!\n");
```

## الخطوة 5: تشغيل البرنامج

```bash
alusus hello.alusus
```

إذا كان كل شيء صحيحًا فستظهر النتيجة التالية في الطرفية:

```text
Hello World!
```

## تشغيل اختياري مع سجل التصحيح

إذا كنت قد بنيت نسخة debug من الوسس من المصدر، فتوثيق البناء يذكر أنه يمكن تفعيل السجل عبر:

```bash
alusus.dbg --log 16 hello.alusus
```

## حل المشاكل الشائعة

### الأمر غير موجود
تأكد من نجاح التثبيت ووجود الرابط التنفيذي.

### مشاكل صلاحيات
استعمل `sudo` أثناء التثبيت فقط، وليس عند تحرير الملفات أو تشغيل البرامج العادية.

### نوع حزمة غير مناسب
استعمل `.deb` مع Debian/Ubuntu و`.rpm` مع توزيعات RPM.

## الخطوات التالية

بعد Hello World:
1. اقرأ دليل المقدمة،
2. تعلّم المتغيرات والسلاسل النصية،
3. جرّب الدوال،
4. ابنِ برنامجًا بسيطًا لسطر الأوامر،
5. ثم انتقل إلى تطبيقات الويب باستخدام WebPlatform.
